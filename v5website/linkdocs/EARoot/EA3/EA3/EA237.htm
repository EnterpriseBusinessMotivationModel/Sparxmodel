A system interaction point is a specific point in a business process where contact occurs between people and systems.  In some methods, a survey of use cases would start with the list of system interaction points that apply to a particular system. <br/><br/>Note that a system interaction can involve more than one system at the same interaction point.  An example would be where a customer is entered into a customer management system, producing the customer id, and then an order is entered into another system referencing the customer id.  <br/><br/>The presence of a system interaction point drives the need for an application feature.  Where a proposed change is driven by a business initiative, that change could potentially drive changes to zero, one, or more system interaction points. Those changes would impact applications.<p/>
